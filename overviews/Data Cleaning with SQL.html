Overview

This project is focused on data cleaning tasks applied to the Nashville Housing database using SQL. The goal of the project is to clean and transform the data to ensure that it is accurate, consistent, and ready for further analysis. Various common data cleaning operations, such as updating date formats, handling NULL values, splitting columns, and removing duplicates, are performed to achieve data integrity and usability.

Key Features

  • Updating Date Format: The project begins by cleaning the SaleDate column to ensure that it stores only the date (excluding the time), making the data more consistent and easier to analyze.
  • Handling NULL Values: NULL values in the PropertyAddress column are handled by using self-joins to fill missing values with corresponding non-null values from other rows within the same ParcelID.
  • Splitting Columns: The PropertyAddress column is split into separate columns for Address and City, and similarly, the OwnerAddress is split into Address, City, and State columns using string manipulation functions.
  • Data Standardization: The SoldAsVacant column values are standardized from numeric (1/0) to descriptive ('Yes'/'No') values for better readability.
  • Removing Duplicates: Duplicate records are identified and removed based on unique attributes such as ParcelID, PropertyAddress, SalePrice, SaleDate, and LegalReference, ensuring that each record is unique.
  • Dropping Redundant Columns: Unnecessary columns like OwnerAddress, TaxDistrict, and others are dropped to simplify the dataset and remove outdated or irrelevant information.

Tools and Technologies Used

  • SQL Server: SQL Server is used to perform data cleaning operations on the NashvilleHousing dataset.
  • SQL Functions: Various SQL functions such as CONVERT(), ISNULL(), SUBSTRING(), CHARINDEX(), PARSENAME(), and ROW_NUMBER() are used to manipulate and clean the data efficiently.
  • Self-Joins: Self-joins are used to fill missing values in the PropertyAddress column by referencing data from other rows with the same ParcelID.
  • Common Table Expressions (CTEs): CTEs are used to handle the identification and removal of duplicate rows by assigning row numbers to grouped records and deleting duplicates based on these row numbers.
  • Data Transformation: The project uses SQL to transform and split columns, standardize values, and drop unnecessary columns, preparing the data for analysis or further processing.

Challenges Solved

  • Handling NULL Values: The challenge of filling NULL values in the PropertyAddress column was solved by using a self-join to find non-null values from other records with the same ParcelID.
  • Data Formatting: Converting the SaleDate column to store only the date (without time) was necessary for consistency, ensuring that the date is usable for time-based analysis.
  • Column Splitting: Complex addresses in the PropertyAddress and OwnerAddress columns were split into multiple columns to facilitate more granular analysis of these fields (e.g., separating city, state, and address).
  • Standardization: Numeric values in the SoldAsVacant column were standardized to 'Yes' and 'No' for better understanding and readability, rather than working with raw numeric values.
  • Removing Duplicates: Identifying and removing duplicate records based on key columns ensured data quality and reduced redundancy, leading to a more accurate dataset for analysis.
  • Efficient Column Management: Dropping redundant columns reduced the dataset's complexity and size, ensuring that only relevant information remains in the final dataset.
